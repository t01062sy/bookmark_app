# セキュリティ方針（利便性優先版）

## 1. 基本原則
- 個人情報は保存しない
- 最大リスクはAPIキー漏えい
- 利便性を損なわない範囲で防御

## 2. 認証
- WebAuthnパスキーのみ
- アクセストークンTTL=15分、リフレッシュ=30日
- デバイス別失効UI

## 3. APIキー管理
- サーバのみ保持、クライアント送信禁止
- 宛先固定（api.openai.com等）
- 30日ごと自動ローテーション
- コスト上限超過時ブレーカー発動

## 4. データ保護
- KMS暗号化（at rest）
- TLS1.2+
- CORSは自オリジンのみ

## 5. やらないこと
- E2EE
- 毎回の2FA
- クライアント直呼び出しのLLM
